# Pastique — development notes

For anyone forking Pastique or wanting to build from source. End-user docs live in [README.md](README.md).

## Why Pastique

You probably already have Maccy, Paste, Raycast Clipboard, or some other clipboard manager. They're all good. Pastique exists because:

- **Truly invisible.** No menubar icon by default after first launch. No Dock entry. No window unless you hit the hotkey.
- **Zero permissions.** Carbon hotkey + NSPasteboard polling. No Accessibility prompt. No Full Disk Access. No login items dialog (until you opt in).
- **Tiny.** ~10MB binary (5MB without Sparkle). One executable, no helper processes, no Electron.
- **Respects sensitive content.** Reads `org.nspasteboard.ConcealedType` / `TransientType` / `AutoGeneratedType`. Your 1Password TOTP never touches disk.

## Features (full list)

- Text, images (PNG / TIFF), files (`file://` URLs from Finder)
- Sort by **recency** or **frequency** (most-used)
- **Compact** (single-line) or **Detailed** (preview + timestamp + use count) view
- Optional **rich-text preservation** — keep RTF/HTML formatting on paste-back
- **Launch at Login** via SMAppService (macOS 13+)
- **Auto-update** via Sparkle 2 (Ed25519 signed)
- Configurable menubar icon (hidden by default after first run)
- Clear all history with one click
- Persists across reboots (SQLite, max 100 items)
- Auto-dedup with use-count tracking

## Build from source

No Xcode required, Command Line Tools is enough:

```sh
xcode-select --install         # ~700MB, only if you don't have it
git clone https://github.com/yiyu0x/pastique.git
cd pastique
./build.sh                     # produces Pastique.app in repo root
open Pastique.app
```

For a universal binary (arm64 + x86_64):

```sh
UNIVERSAL=1 ./build.sh
```

## Tests

```sh
./test.sh
```

30 tests across 4 suites: store CRUD, dedup + use_count, sort orders, image storage, file URLs, trim behavior, rich payload cascade, concealed-type filter. The script auto-detects whether you have full Xcode or only Command Line Tools and adjusts the framework search paths accordingly.

## Architecture

| File | Role |
|---|---|
| `PastiqueApp.swift` | `@main` entry, `NSApplication.setActivationPolicy(.accessory)` |
| `AppDelegate.swift` | Wires watcher, hotkey, picker, menubar; handles all menu actions |
| `ClipboardWatcher.swift` | Background `DispatchSourceTimer` polls `NSPasteboard.changeCount` every 500ms |
| `ClipStore.swift` | GRDB `DatabasePool`, schema migrations, dedup + use-count, image disk storage |
| `HotKeyManager.swift` | Carbon `RegisterEventHotKey` wrapper (no Accessibility needed) |
| `LoginItem.swift` | `SMAppService.mainApp` wrapper |
| `UpdaterController.swift` | Sparkle 2 wrapper with graceful no-op when keys absent |
| `PickerPanel.swift` | `NSPanel` + `NSVisualEffectView` + `NSHostingView`, cursor positioning, paste-back |
| `PickerView.swift` | SwiftUI list, compact/detailed row rendering, hover + click + keyboard |
| `PickerViewModel.swift` | `@MainActor` observable, reload + nav + pick handlers |
| `Settings.swift` | `UserDefaults` wrapper, first-run defaults |

Schema:

```
clips
  id, kind, preview, created_at, text_content, image_path,
  thumbnail (blob), file_urls (json), use_count

clip_payloads        (rich-text mode only, FK CASCADE)
  clip_id, uti, data
```

Images live outside the DB at `images/{uuid}.png`. The DB stores only a relative filename + a 128px PNG thumbnail blob (~5KB). Picker render reads only thumbnails. The full image is read only when the user picks it.

## Privacy & storage

- Everything stays on the user's Mac. No network calls except Sparkle's appcast fetch (https only).
- Data lives at `~/Library/Application Support/Pastique/` — `clips.db` + `images/`.
- Content tagged `org.nspasteboard.ConcealedType` / `TransientType` / `AutoGeneratedType` is never written to disk.
- Two-step paste: Pastique writes the picked item to the system pasteboard, user presses `⌘V`. Auto-paste would need Accessibility permission, which violates the zero-permission goal.

## Auto-update setup (Sparkle)

Releases ship with [Sparkle 2](https://sparkle-project.org) so users get prompted to install new versions automatically. Per fork, do this once:

```sh
tools/setup-sparkle.sh
```

This generates an Ed25519 keypair, prints the public key (paste into `Resources/Info.plist` as `SUPublicEDKey`), and writes the private key to `sparkle_private.key`:

```sh
gh secret set SPARKLE_PRIVATE_KEY < sparkle_private.key
rm sparkle_private.key                    # don't leave it on disk
```

The release workflow commits a signed `appcast.xml` to the `gh-pages` branch. Sparkle fetches it directly from `raw.githubusercontent.com` — no GitHub Pages, custom domain, or external host required. The repo just needs to be public.

If `SUPublicEDKey` is empty or `SPARKLE_PRIVATE_KEY` is unset, the release workflow still publishes a `.zip` to GitHub Releases — it just skips the appcast step so existing users won't auto-update. Useful for previews.

## Release flow

```sh
./test.sh
git commit -am "..."
git push
git tag v0.X.Y
git push origin v0.X.Y      # → release.yml runs ~3-5 min, .zip appears in Releases
```

`release.yml` runs on `macos-15`:
1. Tests (`swift test`)
2. Stamps `CFBundleShortVersionString` (tag minus `v`) and `CFBundleVersion` (`git rev-list --count HEAD`) into Info.plist
3. Universal build (`UNIVERSAL=1 ./build.sh`)
4. `ditto -c -k --sequesterRsrc --keepParent` to zip
5. `sign_update -f <private.key>` for Sparkle
6. Pushes signed entry to `gh-pages/appcast.xml`
7. Creates GitHub Release with the zip

## Icon

The app icon is generated from `tools/make-icon.swift`:

```sh
swift tools/make-icon.swift
# regenerates Resources/AppIcon.iconset and Resources/AppIcon.icns
```

Re-run any time you want to rebrand. `build.sh` automatically bundles the resulting `.icns`.

## Contributing

PRs welcome. Keep it small and boring. The two hard constraints:
1. No permission prompts (no Accessibility, no Full Disk, no Login Items dialog beyond what SMAppService natively shows)
2. Single `.app` bundle, no helper processes you didn't ship with Sparkle
